# 🔑 Revision Keys — Quick-Read Concept Refreshers

**Purpose**: Last-minute interview prep. Padhne mein 5-10 min per day. Saare concepts freshen ho jate hain bina full lesson re-read kiye.

---

## 📂 Structure

```
revision/
├── README.md                          ← you are here
├── day-001-user-registration.md
├── day-002-user-login-jwt.md
├── day-003-user-profile-update.md
├── day-004-password-reset.md
└── day-XXX-topic.md (one per lesson)
```

Same filename as lesson (`day-XXX-topic-slug.md`) — 1:1 mapping.

## 📐 Each File Contains

1. **Link to full lesson** (top)
2. **"If You Remember Only 10 Things"** — absolute essentials (read this if 60 seconds)
3. **Per-Stack Quick Keys** — bullet points (Java, .NET, SQL, Angular, System Design, OOP)
4. **Interview One-Liners** — sentences ready to speak
5. **Mnemonics + Memory Hooks** — TRACE, BARTAN BADLO etc.
6. **Red Flags To Avoid** — what NOT to say

## 🎯 When To Use

| Scenario | Use This File |
|----------|---------------|
| Interview kal hai | Read 5-10 revision files (last 5-10 days covered) |
| Weekly self-check | Last 7 days ke revision files re-scan |
| "Yeh concept kya tha?" | Search revision file (lighter than full lesson) |
| Walking/commute revision | Open mobile, read keys on phone |

## 🔄 Auto-Update

Mentor creates one revision file daily, alongside lesson push. No manual work for you.

## 💡 Tip

If you find yourself **fully recalling** a key from memory → you've mastered it. If you blink at a key → re-open the lesson and re-read that section.

This is your **spaced repetition** anchor.
